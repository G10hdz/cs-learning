## What this is

Static CS/AI self-study roadmap site.
Stack: Vite + React 19 + TypeScript + Tailwind v4 + MDX.

Deploy target: Netlify (`netlify.toml`, SPA redirect).

## Commands

```bash
npm run dev          # Vite dev server → http://localhost:5173 (Turbopack off, plain Vite)
npm run build        # tsc + vite build (prebuild: convert + sync)
npm run lint         # ESLint
npm run seed         # Generate placeholder MDX files (idempotent)
npm run convert      # Obsidian vault → MDX conversion
npm run sync         # Sync NotebookLM content into MDX sync markers
npx tsc --noEmit     # Type-check only
```

## Content frontmatter schema

```ts
type ContentFrontmatter = {
  title: string
  topic: string          // 'math' | 'systems' | 'ai' (drives RoadmapTree grouping)
  difficulty: string     // 'beginner' | 'intermediate' | 'advanced'
  timeEstimate: string   // e.g. '3w', '4w', '6w'
  tags: string[]
  studyTool?: 'notebooklm' | 'ollama' | 'obsidian'
  notebooklmModuleId?: string   // triggers NotebookLM sync
  localSourceZip?: string | string[]
  phase?: number         // 1=Math, 2=Systems+AI, 3=Advanced, 4=Integration
}
```

## Routing

- `/` — Home: sidebar (search + filters + progress backup) + RoadmapTree
- `/roadmap/:slug` — Lazy-loaded MDX page (slug = filename without .mdx, lowercased-hyphenated)
- `*` — 404

Slugs are derived from filename only (not path). `optimization-convexa.mdx` → `/roadmap/optimization-convexa`.

## Virtual module: `virtual:mdx-raw`

Defined in `vite.config.ts` via `mdxRawPlugin`. Reads `.mdx` files with `fs.readFileSync` at build time, bypasses MDX transform. **Do NOT use `import.meta.glob` with `?raw` on `.mdx` files** — MDX rollup plugin intercepts those.

## Key architectural constraints

- MDX files < 5KB each (fast lazy-loading)
- MDX import paths: always `../../components/Foo` (2 levels up from `content/math/`)
- Wiki links: `[[slug]]` → `/roadmap/slug` (via `remarkWikiLinks` in vite.config.ts)
- Comments in MDX: `{/* <!-- comment --> */}` (not bare HTML comments — breaks MDX parser)
- Tailwind v4: uses `@tailwindcss/postcss` plugin, NOT the old `tailwindcss` PostCSS plugin

## NotebookLM sync markers

Place in any MDX file that has `notebooklmModuleId`:
```mdx
{/* <!-- notebooklm-sync-start --> */}
> Content here gets overwritten by `npm run sync`
{/* <!-- notebooklm-sync-end --> */}
```

## Progress system

`useProgress` (Zustand, localStorage key `roadmap-progress`):
- Key format: `moduleId:itemIndex` (e.g. `optimization-convexa:0`)
- `DeliverableList` component handles toggle + progress bar

`useUnifiedProgress` (multi-track Zustand store, localStorage key `unified-progress`):
- Tracks all 5 tracks + gamification state
- XP per difficulty: beginner 10, intermediate 25, advanced 50
- Level formula: `floor(sqrt(totalXP / 100)) + 1`

`useTracker` (learning tracker Zustand store, localStorage key `learning-tracker`):
- Types: book, article, video, podcast, paper, course
- Statuses: want, in_progress, done, dropped
- Quick-save from content pages via floating button
