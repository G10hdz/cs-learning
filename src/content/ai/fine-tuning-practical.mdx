## Proyecto 1: Hypatia Fine-tune (empieza aquí)

### Por qué este primero
- Dataset ya existe: `enhanced_explanations.json` con 6,570 pares Q&A + explicaciones detalladas
- Formato limpio, dominio bien definido (UNAM admisión)
- Métrica clara: accuracy en preguntas del examen
- Resultado útil inmediatamente para Hypatia-public

### Preparar el dataset

```python
import json
import pandas as pd
from datasets import Dataset

# Cargar el dataset de Hypatia
with open('/home/gio/Vscode-projects/Hypatia-public/CubPrep/enhanced_explanations.json') as f:
    data = json.load(f)

# Convertir a formato Alpaca
def to_alpaca(item):
    return {
        "instruction": f"Responde esta pregunta del examen de admisión UNAM (materia: {item.get('materia', 'General')}):\n\n{item['pregunta']}",
        "input": f"Opciones:\n" + "\n".join([f"{chr(65+i)}. {op}" for i, op in enumerate(item['opciones'])]),
        "output": f"Respuesta: {item['respuesta_correcta']}\n\n{item.get('explicacion_correcta', '')}"
    }

alpaca_data = [to_alpaca(item) for item in data if item.get('explicacion_correcta')]
print(f"Dataset: {len(alpaca_data)} ejemplos")

dataset = Dataset.from_list(alpaca_data)
dataset = dataset.train_test_split(test_size=0.1)
```

### Training con Unsloth (tu RX 6700 XT)

```python
from unsloth import FastLanguageModel
from trl import SFTTrainer
from transformers import TrainingArguments

# Cargar modelo base — Qwen2.5-7B es el mejor para español
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="Qwen/Qwen2.5-7B-Instruct",
    max_seq_length=2048,
    dtype=None,           # Auto-detect
    load_in_4bit=True,    # QLoRA — necesario para 12GB VRAM
)

# Aplicar LoRA
model = FastLanguageModel.get_peft_model(
    model,
    r=16,                 # Rank — empieza con 16, sube a 32 si tienes VRAM
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
    lora_alpha=32,
    lora_dropout=0.05,
    bias="none",
    use_gradient_checkpointing="unsloth",  # Reduce VRAM ~30%
)

# Training
trainer = SFTTrainer(
    model=model,
    tokenizer=tokenizer,
    train_dataset=dataset["train"],
    eval_dataset=dataset["test"],
    dataset_text_field="text",   # necesita columna "text" formateada
    max_seq_length=2048,
    args=TrainingArguments(
        per_device_train_batch_size=2,
        gradient_accumulation_steps=4,    # batch efectivo = 8
        warmup_steps=50,
        num_train_epochs=3,
        learning_rate=2e-4,
        fp16=False,                       # ROCm: usa bf16 si disponible
        bf16=True,
        logging_steps=10,
        optim="adamw_8bit",               # Ahorra VRAM
        output_dir="./hypatia-checkpoints",
        evaluation_strategy="steps",
        eval_steps=100,
        save_steps=200,
    ),
)

trainer.train()
model.save_pretrained("./positronica-hypatia-7b")
```

## Evaluación de modelos de dominio

```python
# Benchmark personalizado para Hypatia
def evaluate_hypatia(model, tokenizer, test_questions):
    correct = 0
    for q in test_questions:
        prompt = f"Pregunta UNAM: {q['pregunta']}\nRespuesta:"
        inputs = tokenizer(prompt, return_tensors="pt").to("cuda")
        
        with torch.no_grad():
            output = model.generate(**inputs, max_new_tokens=50)
        
        predicted = tokenizer.decode(output[0], skip_special_tokens=True)
        if q['respuesta_correcta'].lower() in predicted.lower():
            correct += 1
    
    return correct / len(test_questions)
```

## Roadmap de hardware para fine-tuning

### Ahora (RX 6700 XT, 12GB VRAM)
| Modelo | Técnica | VRAM | Tiempo/época |
|---|---|---|---|
| Qwen2.5-1.5B | QLoRA r=16 | ~4GB | ~5 min |
| Qwen2.5-7B | QLoRA r=16 | ~10GB | ~45 min |
| Qwen2.5-14B | ❌ OOM | >12GB | — |

### Futuro (DGX Spark, 128GB unified)
| Modelo | Técnica | VRAM | Tiempo/época |
|---|---|---|---|
| Qwen2.5-7B | Full fine-tune | ~30GB | ~15 min |
| Qwen2.5-14B | LoRA r=64 | ~40GB | ~25 min |
| Qwen2.5-72B | QLoRA r=32 | ~80GB | ~2h |
| Llama3.3-70B | QLoRA r=16 | ~70GB | ~3h |

**Por qué DGX Spark sobre RX 7900 XTX:**
- 128GB unified vs 24GB VRAM — 5x más capacidad
- CUDA = HuggingFace/Unsloth sin fricción ROCm
- 300W TDP vs 355W (RX 7900 XTX) — ligeramente más eficiente para el compute
- Justificado para entrenamiento + inference production
