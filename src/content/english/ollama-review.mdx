# Ollama Review

> **Objetivo:** Escribir una review técnica de Ollama en inglés — documentar setup, modelos probados, performance. El deliverable es una pieza de technical writing real que podrías publicar en un blog o dev.to.

## Objetivos de aprendizaje

- Instalar y configurar Ollama localmente — probar al menos 3 modelos de diferente tamaño
- Escribir una review técnica de 500+ palabras en inglés con estructura profesional
- Usar vocabulario técnico preciso para describir performance, latencia, calidad de output
- Estructurar la review como un blog post: hook → setup → benchmarks → conclusión
- Recibir feedback de NotebookLM sobre claridad, gramática y tono del escrito

## Conceptos clave

### Estructura de una Technical Review

```
1. HOOK (1-2 frases)
   ¿Por qué le interesa esto al lector? Problema que Ollama resuelve.

2. WHAT IS OLLAMA? (3-4 frases)
   "Ollama is an open-source tool that lets you run LLMs locally with one command.
   No cloud, no API keys, no latency. Think Docker, but for LLMs."

3. SETUP EXPERIENCE (1 párrafo)
   ¿Qué tan fácil fue instalar? ¿Sorpresas? ¿Problemas?
   "Installation was straightforward: brew install ollama on macOS, or a single
   binary on Linux. Within 2 minutes I was running my first model."

4. MODELS TESTED (tabla + comentario)
   Lista los modelos, tamaño, y primera impresión de cada uno.

5. PERFORMANCE / BENCHMARKS (datos concretos)
   Tokens/segundo, uso de RAM, calidad de respuesta para un prompt estándar.

6. USE CASES (3-4 bullets)
   ¿Para qué sirve esto realmente? ¿Para qué NO sirve?

7. VERDICT / RECOMMENDATION (2-3 frases)
   ¿Lo recomendarías? ¿A quién? ¿Vale la pena vs ChatGPT/Gemini API?
```

### Technical Writing Vocabulary for Reviews

| Concepto | Frase |
|----------|-------|
| Setup | "Getting started with Ollama is refreshingly simple..." |
| Performance | "On my M1 MacBook with 16GB RAM, llama3.2 generated at ~25 tokens/sec..." |
| Comparison | "Compared to the ChatGPT API, Ollama trades latency for privacy..." |
| Limitations | "The main limitation I encountered was..." / "Where Ollama falls short is..." |
| Surprise | "What surprised me most was..." / "I didn't expect..." |
| Verdict | "For [use case], Ollama is a no-brainer. For [other use case], I'd stick with [alternative]." |

## Estrategia (1 semana)

**Días 1-2 — Setup + exploración**
- Instalar Ollama: `curl -fsSL https://ollama.com/install.sh | sh`
- Pull 3 modelos: `ollama pull llama3.2`, `ollama pull mistral`, `ollama pull qwen2.5-coder`
- Probar cada modelo con el mismo prompt: "Explain backpropagation in simple terms"
- Tomar notas: tiempo de respuesta, calidad, uso de RAM (`ollama ps`)

**Días 3-4 — Benchmarks + escritura (borrador)**
- Definir 3 prompts de prueba estándar (explicación, código, razonamiento)
- Correr los 3 prompts en los 3 modelos — anotar resultados
- Escribir borrador de la review: 300 palabras, estructura completa

**Días 5-6 — Revisión + expansión**
- Pasar borrador por NotebookLM: pedir feedback de claridad, gramática, tono
- Expandir a 500+ palabras — añadir use cases, limitaciones, comparación con API
- Leer en voz alta — ¿suena natural? ¿demasiado formal?

**Día 7 — Versión final + vocabulario**
- Pulir versión final
- Extraer 20 palabras/expresiones técnicas usadas en la review → flashcards
- Quiz NotebookLM sobre vocabulario de la review
